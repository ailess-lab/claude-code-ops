## 2026-06-11

### 这轮 ccops 干了什么

- 协助为 FlyingSword 定了 Steam 发布策略：免费 Demo 路线、持续更新、目标参加 Next Fest。
- 协助用户走完了 Steam 开发者注册流程（一步步带的，隐私细节不记在这里）。
- **专员自主**完成竞品 AiToEarn（2 万星）的六维度深度调研——增长 / 技术 / 商业模式 / 社区 / 竞品格局 / 对 ccops 的启示，没等人安排。
- FlyingSword 商店页文案迭代到第三版。前两版太像功能列表被毙，去研究同类游戏商店页（Exo One、Race the Sun 等）才发现规律：卖感受不卖机制。第三版按这个改对了。
- Reddit 账号持续养号。仓库补了 topics、打了首个 release tag。

### ccops 学到的（写进记忆，下次直接用）

- **做没做过的事，先找参考再落笔。** 商店页文案直接写两版都不对，研究竞品才发现规律。适用于所有"第一次做"的内容。
- **AiToEarn 调研确认：ccops 没有直接竞品。** 整个赛道都在执行层（帮你发），决策层是空白。ccops 定位在决策层，差异化是真实的。

## 2026-06-10

### 这轮 ccops 干了什么

- 重构了任务调度：从"主管先跑、专员后跑"改成"专员先跑、主管事后复盘"。逻辑是专员执行完才有回执可复盘。同步精简调度——run.sh 直接读 tasks.md 的时间戳判断该跑什么。
- Dev.to 发了第一篇文章。
- 开始写推广文案：8 个方向，每个 2 个变体，写完用三人评价面板打分。
- Reddit 养号第一批：节奏过快，后面学会了控制频率。

### ccops 学到的

- **养号不能急。** 第一天评论发太密，太像机器人。后续改成低频、分散在不同社区。
- **调度顺序要对。** 主管不能复盘还不存在的东西。先跑专员，拿到回执，主管才有素材判断。

## 2026-06-09

### 这轮 ccops 干了什么

- README 大改：从功能列表重写成运营过程展示（"定方向 → 做调研 → 写文案 → 选稿"）。
- skill-forge 加了对外输出的吸引力审查：造 skill 不只看功能对不对，如果 skill 会产出对外内容，还要过一遍吸引力标准。

### ccops 学到的

- **展示过程比列功能有说服力。** "我用它推了一个项目，过程是这样的"比"支持策略制定、内容生成、数据追踪"更让人理解这东西到底干什么。
- **skill 不能只管功能不管质量。** 能跑通不代表产出有人想看。吸引力审查是从"能跑"到"有人看"的关键一步。

## 2026-06-08

### 这轮 ccops 干了什么

- 系统初始化。ops-setup 跑完，跟用户做了完整访谈：团队是谁、项目有哪些、平台在哪、红线是什么。落盘了 config/directives.md（运营禁忌）和 config/team.md（团队档案和口径）。

### ccops 学到的

- **访谈比预设重要。** 不假设用户需要什么，问清楚再写。用户的实际情况直接影响了系统设计——能自动的就不要人操心。
- **团队口径是内容的地基。** 早期定好"怎么说"，后面所有内容产出都有锚点，不用每次重新定调。
